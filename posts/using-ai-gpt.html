Practical ways I use AI/GPT as an SRE

2025-10-30 · ai, gpt, sre, productivity

AI copilots accelerate day‑to‑day SRE work when used deliberately. Here are a few workflows that consistently save time without compromising quality.

Tools I use

  • Cursor – IDE-integrated coding assistant for refactors and inline prompts.
  • Windsurf – Agentic workflows for multi-step changes and repo-wide edits.
  • Perplexity – Web research and source-grounded summaries.
  • ChatGPT – Fast iteration for drafts, explanations, and comparisons.

1. Fast Kubernetes iteration

  • Explain events and error messages with concrete next steps.
  • Draft Helm values for common patterns (HPA, PDB, PodSecurity, NetworkPolicy).
  • Summarize logs to identify suspects quickly.

2. Infrastructure as Code

  • Scaffold Terraform modules and examples, then harden manually.
  • Generate drift checklists and pre-commit hooks.

3. Workflows and Automation

  • Draft workflows/pipelines and custom scripts.

Principles

  • Always verify outputs locally and in staging.
  • Never paste secrets; use redacted snippets.
  • Iterate with short prompts and concrete artifacts.
